## 3. 종합 결론

**이 프로젝트의 에어갭 "이식" 자체는 쉽고, 작업량은 "재무장"에 집중됩니다.** 원본 도구의 정체성은 공식 Claude in Chrome의 제약(58개 도메인 블록리스트·계획 승인·권한 게이트)을 *걷어낸* 것입니다. 그러나 사내 폐쇄망 엔터프라이즈는 정반대로 **접근통제·승인·감사**를 요구합니다. 따라서 커스터마이징의 본질은 "기능 추가"가 아니라 **제거된 안전장치의 선별적 복원**입니다.

순수 이식만으로 "동작"은 하지만, 다수 사용자 폐쇄망에서 **안전하게 운영**하려면 보안 High 항목(로컬 인증·도메인 화이트리스트·승인 게이트·`javascript_tool` 통제·감사 로깅)이 **도입 전제 조건**입니다. LLM은 LiteLLM+vLLM 경로가 에어갭에서 유일하게 성립하며, 유일한 본질적 불확실성인 **오픈모델 툴콜 안정성**은 Phase 1 PoC에서 조기 판별 후 대안(다른 모델/비전 모델/텍스트-퍼스트)으로 대응 가능합니다.

## 4. 대상 시스템 구조

```
Claude Code ──stdio MCP──▶ mcp-server.js ──TCP :18765──▶ native-host.js ──native messaging──▶ 확장(background.js) ──CDP──▶ 브라우저 탭
   (LLM 두뇌)                  (MCP 18툴)        (로컬 루프백)        (브리지)                  (자동화 실행)

별도 경로:  Claude Code ──ANTHROPIC_BASE_URL──▶ [내부] LiteLLM Proxy ──▶ vLLM(오픈모델 풀, GPU 노드)
```

- 18개 MCP 툴 중 **실동작 12개** + **스텁 6개**(`gif_creator`/`shortcuts_*`/`switch_browser`/`upload_image`(부분)/`update_plan`(자동승인)).
- 모든 내부 통신은 `127.0.0.1` 루프백 → 외부 인터페이스 미노출(이식·보안에 유리, 단 무인증).

## 5. 영역별 핵심 요약

### ① LLM 모델 & 서빙 인프라 → [01 문서](./01-LLM-모델-및-서빙-인프라.md)
- **연결**: `ANTHROPIC_BASE_URL` → **LiteLLM `/v1/messages` unified** → **vLLM**. 에어갭에서 Bedrock/Vertex 불가하므로 유일 성립 경로.
- **메인 모델**: **Qwen3-Coder 30B-A3B (FP8)** — Apache-2.0(법무 안전), MoE 활성 3B로 H100 1~2장 고동시성, 256K 컨텍스트, 전용 툴콜 파서. **고난도(옵션)**: DeepSeek-V3(H100×8 점유).
- **비전**: `computer:screenshot`/`zoom`이 JPEG 반환 → 비전 모델(Qwen2.5-VL) 쓰면 공식 경험 동등, 텍스트 모델만 쓰면 `read_page`/`find`/`ref` **텍스트-퍼스트** 우회.
- **호환성 위험**: 병렬 툴콜 약함, tool_use/tool_result 순서 제약, reasoning 모델 `<think>` 내 툴콜 유실, count_tokens 오차 → **PoC 실측 필수**.
- **양자화**: H100이면 **FP8** 기본. INT4는 동시성 극대화 때만(긴 툴콜 루프 정확도 주의).

### ② 아키텍처 & 의존성 오프라인화 → [02 문서](./02-아키텍처-및-의존성-오프라인화.md)
- **외부 의존은 설치 시점에만**: 직접 의존 `@modelcontextprotocol/sdk` 1개 + 전이 91개(전부 순수 JS, 네이티브 빌드 없음). 런타임 외부망 코드 경로 **없음**.
- **오프라인화 1순위**: `node_modules` **vendoring**(외부 PC `npm ci` → `.tgz` 반입). 빌드 스텝 0이라 폴더 복사만으로 OS 무관 배포.
- **연결은 코드 변경 없이 설정만**: `ANTHROPIC_BASE_URL`·`config.json` 포트·`.mcp.json` 등록.
- **Windows 보완**: `native-host-wrapper.bat` + `install.ps1`(레지스트리 등록) 신규 작성(현 `install.sh`는 mac/Linux 전용).
- **코드 변경 지점**: 화이트리스트/감사 로깅(`navigate`), 비전 미지원 시 이미지→텍스트 대체(`computer`), 사내 네이밍(호스트명/포트 다지점 동기화).

### ③ 보안 & 거버넌스 → [03 문서](./03-보안-및-거버넌스.md)
- **위험 본질**: 에어갭이 *외부 유출*은 막지만, 실질 위협은 **내부 자산(인사/회계/관리자 콘솔)에 대한 무인·무인증·무감사 제어**.
- **High 결함 4종**: ⓐTCP:18765 **무인증**(동일 PC 임의 프로세스가 `client_hello` 한 줄로 제어권 탈취) ⓑ`update_plan` **자동승인**(권한 게이트 부재) ⓒ`javascript_tool` 임의 JS 실행 ⓓ도메인 **화이트리스트·감사 로그 전무**.
- **프롬프트 인젝션**: 신뢰 못 할 내부 페이지 콘텐츠가 LLM에 주입 → 쓰기성 도구 트리거 가능.
- **권고**: 로컬 핸드셰이크 인증 / 내부 화이트리스트 / 승인 게이트 복원 / `javascript_tool` 통제 / append-only 감사 로깅 (모두 **High = 운영 전 필수**).

### ④ 배포 & 운영 → [04 문서](./04-배포-및-운영.md)
- **본 환경은 개발자 모드 허용 → 셀프서비스(Load unpacked + 사용자별 install)가 기본 경로(Path A).** `install.sh`가 확장 ID를 인자로 받아 PC별 ID 상이도 무방.
- **중앙 강제배포(Path B, 옵션)**: 비기술 사용자 확대·자동 업데이트·무결성 중앙 보증·개발자 모드 차단 시 → `key` 고정 + `.crx` + `ExtensionInstallForcelist`.
- **Windows**: 레지스트리(HKLM) 등록 + **Node 동봉** MSI(Intune/SCCM/GPO 배포).
- **멀티세션**: primary/client 멀티플렉싱은 **단일 PC 내** 공유. 포트 18765는 PC-로컬이라 **PC 간 충돌 없음**, 단 **PC당 단일 브라우저 프로필**·**1인1세션 PC** 권장(공용 멀티세션 PC 비권장).
- **운영**: 4계층 버전 관리·링 배포·롤백, 합성 모니터(reddit 스모크 + LLM ping), 장애 런북.

## 6. 교차검증 — 영역 간 종합 판정

| 교차 이슈 | 발견 | 종합 판정 |
