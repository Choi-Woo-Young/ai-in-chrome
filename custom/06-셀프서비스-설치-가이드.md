# 셀프서비스 설치 가이드 (에어갭 VDI · Windows 우선)

> 하네스팀 검토 ⑥ — 배포 방식 **셀프서비스 확정**에 따른 실제 설치 런북
> 전제: 완전 에어갭 · 이미 운영 중인 사용자별 VDI · 개발자 모드 허용 · 두뇌=opencode · 모델=중앙 vLLM
> 상위 맥락은 [05 문서](./05-확정-아키텍처-opencode-및-PoC.md) 참조.

---

## 0. 역할 분담

| 주체 | 작업 |
|---|---|
| **IT(중앙, 1회)** | ① 반입 번들 제작(아래 §1) ② 사내 공유 폴더/매체에 배치 ③ 중앙 vLLM(gpt-oss/Qwen3.5-27B/VL) 운영 |
| **각 사용자(자기 VDI, 1회)** | 번들 받아 §2 절차대로 설치(스크립트 1회 + Load unpacked 1회) |

> ✅ **opencode는 대부분의 VDI에 이미 설치·사용 중**(터미널 또는 데스크톱 앱). 이 경우 **기존 opencode에 설정(`opencode.json`)만 주입**합니다.
>
> ⚠️ **단, 미설치 사용자도 있을 수 있으므로 번들에 opencode를 동봉**합니다. `install.ps1`이 **기존 설치를 자동 감지** → 있으면 설정만 주입, 없으면 **동봉본을 설치**(폴백). 즉 "있으면 재사용, 없으면 설치"가 한 스크립트로 처리됩니다.
>
> ⚠️ **설치 방식(npm/CLI vs 데스크톱 앱)은 우리 통합과 무관** — 그 opencode가 ① 로컬(stdio) MCP 서버 지원 ② 커스텀 OpenAI 호환 프로바이더 ③ `~/.config/opencode/opencode.json` 공유, 세 가지를 충족하면 됩니다. **터미널(CLI)은 거의 확실히 충족**, **데스크톱 앱은 "로컬 stdio MCP spawn + 설정 경로"를 PoC에서 1회 확인**(샌드박싱 시 제한 가능). node 절대경로를 설정에 박아 PATH 의존을 제거(§3).

## 1. IT가 1회 준비하는 반입 번들

외부망 PC에서 만들어 **무결성 검증(SHA-256) 후 에어갭 반입** → 사내 공유 폴더에 배치.

### 1.1 번들 레이아웃
```
OpenClaude-bundle/
├─ extension/                 # MV3 확장 (이 레포 extension/ 그대로)
├─ host/
│   ├─ mcp-server.js
│   ├─ native-host.js
│   ├─ package.json
│   └─ node_modules/          # vendored (외부 PC에서 `npm ci` 후 동봉, 02 문서 §2.2)
├─ node/                      # portable Node v20 (node.exe 등) — PC에 Node 없을 수 있으므로 동봉
├─ opencode/                  # opencode 동봉본(폴백) — 미설치 사용자용. 기존 설치 있으면 미사용
├─ config/
│   ├─ opencode.json          # vLLM 엔드포인트·MCP 등록 템플릿 (05 문서 §2.2)
│   └─ AGENTS.md              # 텍스트-퍼스트 규칙
└─ install.ps1                # 사용자 실행 스크립트 (§3)
```

### 1.2 번들 만들기 (외부망 PC, 1회)
```bash
# host 의존성 vendoring
cd host && npm ci && cd ..
# 번들 폴더 구성(개념) — extension/ host/ node/ opencode/ config/ install.ps1 모으기
# portable Node·opencode(폴백용)는 공식 배포에서 받아 node/ , opencode/ 에 배치
# 무결성 해시 생성
#   (Windows 반입 후 대조용) certutil -hashfile <file> SHA256
```
> ⚠️ `node_modules`·Node·opencode·모델 가중치는 **해시 + 서명 검증** 후 반입(03 문서 §5). opencode 동봉본은 **미설치 사용자용 폴백**이며, 기존 설치가 있으면 install.ps1이 건드리지 않음.

## 2. 사용자 설치 절차 (각 VDI에서 1회)

> Windows 기준(사내 표준 가정). mac/Linux는 §5.

1. **번들 복사**: 공유 폴더의 `OpenClaude-bundle`을 고정 경로로 복사 →
   `%LOCALAPPDATA%\OpenClaude` (관리자 권한 불필요)
   - **중요**: 경로를 **모든 사용자가 동일하게** 쓰면 확장 ID가 전 VDI 일관(05 §5.3).
2. **확장 로드(개발자 모드)**: 브라우저에서 `chrome://extensions`(또는 `edge://extensions`) → **개발자 모드 ON** → **압축해제된 확장 프로그램을 로드** → `%LOCALAPPDATA%\OpenClaude\extension` 선택 → **표시된 확장 ID 복사**.
3. **설치 스크립트 실행**(PowerShell):
   ```powershell
   cd $env:LOCALAPPDATA\OpenClaude
   .\install.ps1 -ExtensionId <2단계에서_복사한_ID> -VllmBaseUrl "http://vllm.internal:8000/v1"
   ```
   → 네이티브 호스트 래퍼·매니페스트 생성 + HKCU 레지스트리 등록 + opencode 설정 배치.
4. **브라우저 완전 재시작**(모든 창 종료 후 다시 열기 — 네이티브 매니페스트 로드).
5. **opencode 실행**: 기존 설치가 있으면 그대로 실행(설정이 `~/.config/opencode/opencode.json`에 주입됨). **미설치였다면** install.ps1이 동봉본을 깔았으므로 **새 터미널을 열어**(PATH 반영) `opencode` 실행.
6. **검증**(§4 스모크 테스트).

> 2~3단계 순서 주의: **먼저 Load unpacked로 ID를 얻고**, 그 ID로 install.ps1을 실행합니다(네이티브 호스트 `allowed_origins`에 ID가 박혀야 연결됨).

## 3. `install.ps1` (Windows 셀프서비스, HKCU·무관리자)

```powershell
# install.ps1 — OpenClaude 셀프서비스 설치 (에어갭 VDI, 관리자 권한 불필요)
# 사용법: .\install.ps1 -ExtensionId <확장ID> [-VllmBaseUrl <url>]
param(
  [Parameter(Mandatory=$true)][string]$ExtensionId,
  [string]$VllmBaseUrl = "http://vllm.internal:8000/v1"
)
$ErrorActionPreference = "Stop"

$Root     = $PSScriptRoot                                  # = %LOCALAPPDATA%\OpenClaude
$HostDir  = Join-Path $Root "host"
$NodeExe  = Join-Path $Root "node\node.exe"
$HostName = "com.anthropic.open_claude_in_chrome"
$Wrapper  = Join-Path $HostDir "native-host-wrapper.cmd"
$Manifest = Join-Path $HostDir "$HostName.json"

# 0) 사전 점검
if (-not (Test-Path $NodeExe))                      { throw "node 미동봉: $NodeExe" }
if (-not (Test-Path (Join-Path $HostDir "node_modules"))) { throw "host\node_modules 없음(반입 누락)" }

# 0.5) opencode 자동 감지 — 있으면 재사용, 없으면 동봉본 설치(폴백)
$ocCmd = Get-Command opencode -ErrorAction SilentlyContinue
if ($ocCmd) {
  Write-Host "[OK] 기존 opencode 발견: $($ocCmd.Source) → 설정만 주입"
} else {
  $bundledOc = Join-Path $Root "opencode"
  if (-not (Test-Path $bundledOc)) { throw "opencode 미설치 + 동봉본 없음: $bundledOc" }
  # 동봉본을 사용자 경로에 배치하고 PATH(사용자 범위)에 추가 — 관리자 불필요
  $ocTarget = Join-Path $env:LOCALAPPDATA "Programs\opencode"
  New-Item -ItemType Directory -Force -Path $ocTarget | Out-Null
  Copy-Item "$bundledOc\*" $ocTarget -Recurse -Force
  $userPath = [Environment]::GetEnvironmentVariable("Path", "User")
  if ($userPath -notlike "*$ocTarget*") {
    [Environment]::SetEnvironmentVariable("Path", "$userPath;$ocTarget", "User")
  }
  Write-Host "[OK] opencode 미설치 → 동봉본 설치: $ocTarget (새 터미널부터 PATH 적용)"
}

# 1) 네이티브 호스트 래퍼(.cmd) — 동봉 node로 native-host.js 실행 + 로그 리다이렉트
$logDir = Join-Path $env:LOCALAPPDATA "OpenClaude\logs"
New-Item -ItemType Directory -Force -Path $logDir | Out-Null
@"
@echo off
"$NodeExe" "$HostDir\native-host.js" >> "$logDir\native-host.log" 2>&1
"@ | Set-Content -Encoding ASCII $Wrapper

# 2) 네이티브 메시징 매니페스트 (allowed_origins = 사용자 확장 ID)
$manifestJson = @"
{
  "name": "$HostName",
  "description": "Open Claude in Chrome Native Messaging Host",
  "path": "$($Wrapper -replace '\\','\\')",
  "type": "stdio",
  "allowed_origins": [ "chrome-extension://$ExtensionId/" ]
}
"@
$manifestJson | Set-Content -Encoding UTF8 $Manifest

# 3) HKCU 네이티브 호스트 등록 (Chrome + Edge) — 관리자 불필요
$targets = @{
  "Chrome" = "HKCU:\Software\Google\Chrome\NativeMessagingHosts\$HostName";
  "Edge"   = "HKCU:\Software\Microsoft\Edge\NativeMessagingHosts\$HostName";
}
foreach ($name in $targets.Keys) {
  New-Item -Path $targets[$name] -Force | Out-Null
  Set-ItemProperty -Path $targets[$name] -Name "(default)" -Value $Manifest
  Write-Host "[OK] native host 등록: $name"
}

# 4) opencode 전역 설정 배치 (~\.config\opencode\opencode.json)
$ocDir = Join-Path $env:USERPROFILE ".config\opencode"
New-Item -ItemType Directory -Force -Path $ocDir | Out-Null
$mcpServerPath = ($HostDir + "\mcp-server.js") -replace '\\','\\'
$ocJson = @"
{
  "provider": {
    "local-vllm": {
      "npm": "@ai-sdk/openai-compatible",
      "options": { "baseURL": "$VllmBaseUrl", "apiKey": "dummy" },
      "models": {
        "gpt-oss":     { "name": "gpt-oss" },
        "qwen3.5-27b": { "name": "Qwen/Qwen3.5-27B" }
      }
    }
  },
  "mcp": {
    "open-claude-in-chrome": {
      "type": "local",
      "command": ["$($NodeExe -replace '\\','\\')", "$mcpServerPath"],
      "enabled": true
    }
  }
}
"@
$ocJson | Set-Content -Encoding UTF8 (Join-Path $ocDir "opencode.json")
Copy-Item (Join-Path $Root "config\AGENTS.md") (Join-Path $ocDir "AGENTS.md") -Force -ErrorAction SilentlyContinue
Write-Host "[OK] opencode 설정: $ocDir\opencode.json (vLLM=$VllmBaseUrl)"

Write-Host "`n완료. 다음:"
Write-Host "  1) 브라우저 완전 재시작(모든 창 종료)"
Write-Host "  2) opencode 실행 후 스모크 테스트(06 문서 §4)"
```

> ⚠️ opencode 설정 키(`provider`/`mcp`, AI SDK 프로바이더명)는 **반입 opencode 버전 문서로 정확히 확인**(05 §2.2 주의). 위는 개념 템플릿입니다.

## 4. 검증 (스모크 테스트)

설치 후 각 VDI에서:

1. `chrome://extensions` → 확장 **활성** + ID가 install.ps1에 넣은 값과 동일.
2. opencode 실행 → 모델 목록에 `local-vllm/gpt-oss` 등 노출(외부 models.dev 조회 없이).
3. opencode에 지시: **"내부 위키(wiki.corp.local)로 이동해서 read_page로 제목을 읽어줘"** →
   - `tabs_context_mcp` → `navigate` → `read_page`(텍스트-퍼스트) 동작 확인.
4. 감사 로그 생성 확인: `%LOCALAPPDATA%\OpenClaude\logs\` 및 `~\.config\open-claude-in-chrome\logs\audit-*.jsonl`.
5. (승인 게이트 적용 시) 미승인 도메인 거부 → `update_plan` 승인 후 자동 실행 확인.

## 5. mac/Linux 사용자 (해당 시)

- `install.ps1` 대신 **`install.sh <확장ID>`**(레포 기존 스크립트, 02 문서 §5 참고) 사용.
- 네이티브 매니페스트는 디렉터리 방식(레지스트리 아님): `~/Library/Application Support/Google/Chrome/NativeMessagingHosts/` 등.
- opencode 설정 경로는 동일(`~/.config/opencode/opencode.json`).

## 6. ⚠️ 반입 전 반드시 확인할 것

| # | 확인 | 이유 |
|---|---|---|
| 1 | **opencode 형태별 MCP 지원**(CLI vs 데스크톱) | 설치 방식(npm/CLI vs 데스크톱 앱)은 무관하나, **데스크톱 앱은 샌드박싱으로 로컬(stdio) MCP spawn이 제한**되거나 **설정 경로가 다를** 수 있음. **CLI를 1순위 검증**, 데스크톱은 "로컬 MCP 서버 + 설정 경로"를 PoC에서 확인. (Windows 네이티브 미지원 버전이면 WSL 경유 가능성도 점검) |
| 2 | opencode 설정 스키마(`provider`/`mcp`) | 버전별 키 차이 — 실제 문서로 대조 |
| 3 | models.dev 외부조회 차단 | 커스텀 프로바이더만으로 동작하는지(외부 호출 0) 확인 |
| 4 | 확장 ID 일관성 | 전 VDI 동일 고정 경로(`%LOCALAPPDATA%\OpenClaude\extension`) 사용 시 ID 동일한지 |
| 5 | vLLM 엔드포인트 도달성 | VDI→중앙 vLLM 네트워크 경로·방화벽 |

## 7. 트러블슈팅 (요약)

| 증상 | 조치 |
|---|---|
| "Browser extension is not connected" | 웹페이지 열어 서비스워커 깨우기 / 브라우저 완전 재시작 / 매니페스트 경로·ID 확인 |
| 네이티브 호스트 미연결 | `%LOCALAPPDATA%\OpenClaude\logs\native-host.log` 확인, HKCU 키 존재·경로 확인 |
| opencode가 모델 못 찾음 | `~/.config/opencode/opencode.json`의 baseURL/모델명, vLLM 도달성 확인 |
| opencode가 Windows에서 안 뜸 | §6-1 — WSL 경유 필요 여부 확인 |
| 포트 18765 점유 | 이전 mcp-server 잔존 프로세스 종료 후 재시도(04 문서 §6) |
